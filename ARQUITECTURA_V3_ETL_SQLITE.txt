PASO 1: ETL Automático (cron 3×/día: 06:30, 12:30, 20:30)
┌─────────────────────────────────────────────────────────────────┐
│ etl/etl_xm_to_sqlite.py:                                        │
│   1. Consultar API XM (pydataxm)                                │
│   2. Convertir unidades (Wh→GWh, kWh→GWh, horas→días)          │
│   3. Validar datos (completitud, rangos)                        │
│   4. Insertar en SQLite (upsert_metrics_bulk)                   │
│                                                                  │
│ 17 métricas procesadas:                                         │
│   • Generación: Gene/Sistema, Gene/Recurso                      │
│   • Embalses: VoluUtilDiarEner, CapaUtilDiarEner               │
│   • Aportes: AporEner, AporEnerMediHist                        │
│   • Hidrología: AporCaudal, PorcApor (por río)                 │
│   • Distribución: DemaCome                                      │
│                                                                  │
│ Resultado: portal_energetico.db actualizada                     │
│            (INSERT OR REPLACE previene duplicados)              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
PASO 2: Usuario accede a Dashboard
┌─────────────────────────────────────────────────────────────────┐
│ app.py (Dash/Flask):                                            │
│   1. Usuario solicita página /generacion                        │
│   2. Callback ejecuta obtener_metricas_hidricas()               │
│   3. Función llama obtener_datos_desde_sqlite()                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
PASO 3: Consulta SQL Directa
┌─────────────────────────────────────────────────────────────────┐
│ utils/_xm.py - obtener_datos_desde_sqlite():                   │
│                                                                  │
│   # Fallback automático (busca 7 días hacia atrás)             │
│   for dias_atras in range(7):                                   │
│       fecha = fecha_fin - timedelta(days=dias_atras)            │
│       df = db_manager.get_metric_data(                          │
│           metrica='Gene',                                       │
│           entidad='Sistema',                                    │
│           fecha_inicio=fecha,                                   │
│           fecha_fin=fecha                                       │
│       )                                                         │
│       if df is not None and not df.empty:                       │
│           return df, fecha  # ✅ Datos encontrados              │
│                                                                  │
│   return None, None  # No hay datos                             │
└─────────────────────────────────────────────────────────────────┘
                              ↓

│   AND entidad = ?                                               │

1. BASE DE DATOS (portal_energetico.db)
┌─────────────────────────────────────────────────────────────────┐
│ Schema (sql/schema.sql):                                        │
│                                                                  │
│ CREATE TABLE metrics (                                          │
│     id INTEGER PRIMARY KEY AUTOINCREMENT,                       │
│     fecha DATE NOT NULL,                                        │
│     metrica VARCHAR(50) NOT NULL,                               │
│     entidad VARCHAR(100) NOT NULL,                              │
│     recurso VARCHAR(100),                                       │
│     valor_gwh REAL NOT NULL,                                    │
│     unidad VARCHAR(10) DEFAULT 'GWh',                           │
│     fecha_actualizacion TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    │
│     UNIQUE(fecha, metrica, entidad, recurso)                    │
│ );                                                              │
│                                                                  │
│ Índices optimizados (5 total):                                  │
│   • idx_fecha: Para búsquedas por fecha                         │
│   • idx_metrica_entidad: Para filtrar por tipo                  │
│   • idx_fecha_metrica: Búsquedas combinadas                     │
│   • idx_fecha_metrica_entidad: Filtros específicos              │
│   • idx_recurso: Para Gene/Recurso (por fuente)                 │
│                                                                  │
│ Capacidad: 281 TB (límite SQLite)                               │
│ Uso actual: ~60 KB (293 registros únicos)                       │
│ Proyección 10 años: ~36 GB                                      │
│ Performance: Queries <5ms                                       │
└─────────────────────────────────────────────────────────────────┘

2. ETL SCRIPT (etl/etl_xm_to_sqlite.py)
┌─────────────────────────────────────────────────────────────────┐
│ Funciones principales:                                          │
│                                                                  │
│ • convertir_unidades(df, metric, conversion_type)               │
│   Conversiones:                                                 │
│   - Wh_a_GWh: valor ÷ 1,000,000                                │
│   - kWh_a_GWh: valor ÷ 1,000,000                               │
│   - horas_a_diario: sum(Hour01-24) ÷ 1,000,000                 │
│                                                                  │
│ • poblar_metrica(obj_api, config, usar_timeout, timeout_sec)    │
│   - Consulta API XM con batching inteligente                    │
│   - Convierte unidades automáticamente                          │
│   - Inserta en SQLite con bulk upsert                           │
│                                                                  │
│ • ejecutar_etl(usar_timeout=True)                               │
│   - Loop sobre METRICAS_CONFIG (17 métricas)                    │
│   - Logging detallado de progreso                               │
│   - Error handling (continúa si una falla)                      │
│   - Retorna estadísticas de ejecución                           │
│                                                                  │
│ Ejecución típica: 50-60 segundos para 17 métricas               │
└─────────────────────────────────────────────────────────────────┘

3. DATABASE MANAGER (utils/db_manager.py)
┌─────────────────────────────────────────────────────────────────┐
│ API Python ↔ SQLite:                                            │
│                                                                  │
│ • get_connection() → Context manager seguro                     │
│ • init_database() → Crea tablas e índices                       │
│ • get_metric_data(metrica, entidad, fecha_ini, fecha_fin, rec)  │
│   → Retorna DataFrame de pandas                                 │
│ • upsert_metric(fecha, metrica, entidad, recurso, valor, unid)  │
│   → Inserta/actualiza 1 registro                                │
│ • upsert_metrics_bulk(lista_tuplas)                             │
│   → Inserta múltiples (más rápido)                              │
│ • get_latest_date(metrica, entidad)                             │
│   → Última fecha con datos                                      │
│ • get_database_stats()                                          │
│   → total_registros, metricas_unicas, fecha_min/max, tamaño_mb │
│ • test_connection() → True/False                                │
│                                                                  │
│ Todas las operaciones con transacciones y error handling        │
└─────────────────────────────────────────────────────────────────┘

4. HELPER XM (utils/_xm.py)
┌─────────────────────────────────────────────────────────────────┐
│ Función nueva (reemplaza obtener_datos_con_fallback):          │
│                                                                  │
│ • obtener_datos_desde_sqlite(metric, entity, fecha_fin,         │
│                               dias_busqueda=7, recurso=None)    │
│   - Fallback automático: busca hasta 7 días atrás              │
│   - Consulta SQLite vía db_manager                              │
│   - Renombra columnas para compatibilidad:                      │
│     • valor_gwh → Value                                         │
│     • fecha → Date                                              │
│     • recurso → Resources/Embalse/Rio/Agente (según entidad)   │
│   - Retorna (DataFrame, fecha_encontrada) o (None, None)        │
│                                                                  │
│ Ventajas vs obtener_datos_con_fallback:                         │
│   ✅ Sin cache_keys frágiles                                    │
│   ✅ Sin lógica de conversión doble                             │
│   ✅ Consulta directa y simple                                  │
│   ✅ Performance <5ms                                            │
└─────────────────────────────────────────────────────────────────┘
